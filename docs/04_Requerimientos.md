# Requerimientos del Sistema

Los requerimientos del sistema describen las funcionalidades y características que deberá cumplir la plataforma para satisfacer las necesidades del proceso de administración del consumo de agua del condominio.

Los requerimientos se dividen en:

- Requerimientos Funcionales (RF): describen las funcionalidades que el sistema debe realizar.
- Requerimientos No Funcionales (RNF): describen las características de calidad, seguridad, rendimiento y restricciones del sistema.


## Requerimientos Funcionales

### RF-001

El sistema deberá permitir el inicio de sesión mediante usuario y contraseña.

---

### RF-002

El sistema deberá administrar diferentes tipos de usuarios.

---

### RF-003

El sistema deberá permitir registrar nuevas viviendas.

---

### RF-004

El sistema deberá permitir modificar la información de una vivienda.

---

### RF-005

El sistema deberá permitir eliminar viviendas.

---

### RF-006

El sistema deberá consultar periódicamente el correo electrónico oficial del condominio.

---

### RF-007

El sistema deberá descargar automáticamente la factura del proveedor.

---

### RF-008

El sistema deberá almacenar la factura en la nube.

---

### RF-009

El sistema deberá extraer automáticamente la información contenida en la factura.

---

### RF-010

El sistema deberá registrar la información de la factura en la base de datos.

---

### RF-011

El sistema deberá permitir cargar fotografías de los medidores.

---

### RF-012

El sistema deberá ejecutar OCR sobre las fotografías.

---

### RF-013

El administrador deberá poder corregir manualmente el resultado del OCR.

---

### RF-014

El sistema deberá calcular automáticamente el consumo mensual.

---

### RF-015

El sistema deberá calcular automáticamente el monto correspondiente a cada vivienda.

---

### RF-016

El sistema deberá generar automáticamente un PDF por vivienda.

---

### RF-017

El sistema deberá almacenar los recibos generados.

---

### RF-018

El sistema deberá enviar automáticamente los recibos mediante WhatsApp.

---

### RF-019

El sistema deberá enviar automáticamente los recibos mediante correo electrónico.

---

### RF-020

El sistema deberá registrar el estado de cada envío.

---

### RF-021

El sistema deberá permitir consultar el historial de consumos.

---

### RF-022

El sistema deberá permitir consultar el historial de recibos.

---

### RF-023

El sistema deberá registrar todas las operaciones importantes del sistema.

---

## Requerimientos No Funcionales

### RNF-001

El sistema deberá estar disponible mediante una aplicación web.

---

### RNF-002

El acceso deberá realizarse mediante autenticación.

---

### RNF-003

La información deberá almacenarse de forma persistente.

---

### RNF-004

El sistema deberá permitir futuras ampliaciones sin modificar su arquitectura principal.

---

### RNF-005

El sistema deberá mantener la integridad de la información.

---

### RNF-006

El sistema deberá registrar errores durante los procesos automáticos.

---

### RNF-007

El sistema deberá proteger la información mediante diferentes niveles de permisos.

---

### RNF-008

El sistema deberá ser compatible con dispositivos móviles y computadoras.
